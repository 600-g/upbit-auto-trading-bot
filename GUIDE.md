2. "Open API 관리" 클릭
3. "API 키 생성" 버튼 클릭
4. **Access Key** 복사
5. **Secret Key** 복사 (보안!)

- API 설정 화면에서 IP 제한 추가

- 예: 192.168.0.30

## 💡 3. 기능 설명

### 3.1 자동 코인 선택
- 5개 메인 코인 (BTC, ETH, SOL, ADA, XRP) 모니터링
- 모멘텀 점수 기반 상위 2개 선택
- **카테고리 다른 것만** 선택
  - BTC + ETH (O)
  - SOL + ADA (O)
  - BTC + SOL (X - 같은 카테고리 아님, 가능)

### 3.2 모멘텀 감지 (0~10점)
계산 방식:
- **거래량 (25%)**: 1시간 vs 24시간 평균
- **RSI (25%)**: 14시간 기반 상대강도지수
- **지지선/저항선 (15%)**: 24시간 고/저점 근처
- **가격 상승률 (15%)**: 최근 15분 변화율
- **차트패턴 (10%)**: 플래그, 웨지, H&S 등
- **뉴스/SNS (10%)**: 뉴스 API + SNS 감시

**점수 해석:**
```
9~10점: 극강 신호 (10% 익절)
7~9점: 강함 (8% 익절)
5~7점: 중간 (6% 익절)
3~5점: 약함 (5% 익절)
0~3점: 매우 약함 (5% 즉시)
```

### 3.3 약세장 동적 임계값
시장 전체가 약할 때 자동으로 거래 기준 상향:

```
극약세 (전체 -15%+): 거래 중단 (9점 이상만)
심약세 (전체 -10%~-15%): 8점 이상만
약세 (전체 -5%~-10%): 7점 이상만
보통 (전체 -3%~+3%): 5점 이상
강세 (전체 +3%+): 4점 이상
```

### 3.4 분할 매수
- 1차 매수: 할당 자금의 25%
- 대기: 5분
- 2차 매수: 할당 자금의 25%

**장점:**
- 저점 매입 가능성 높음
- 평균 단가 낮춤

### 3.5 자동 익절/손절

**익절:**
- 모멘텀 기반 목표 달성 시 자동 매도
- 배치별 독립 관리

**손절:**
- -5% 손실 시 자동 손절
- 극단 변동 감지 시 즉시 손절

### 3.6 극단 변동 감지
- 1분에 5% 이상 급동 → 거래 정지
- 3분에 10% 이상 급동 → 자동 손절
- 거래량 급감 감지 → 거래 금지 (유동성 부족)

## 🎮 4. 명령어

### 거래 명령어

```
run              거래 사이클 시작 (코인 선택 → 매수 → 모니터링)
status           현재 포지션 및 수익률 확인
save             로그 저장
exit             프로그램 종료
```

### 사용 예시

```
> run
[2026-02-20 01:30:00] === 거래 사이클 시작 ===

[매수] BTC 250000원 @ 70000000원

> status
=== 상태 ===
현재 잔액: 4,500,000원
보유 포지션: 2
  BTC batch_1: +1.50%
  BTC batch_2: +0.80%

> save
로그 저장됨: trading_log_20260220_013000.json

> exit
프로그램 종료
```

## ⚠️ 5. 주의사항

### 5.1 보안
- ❌ API 키를 코드에 직접 저장 금지
- ✅ 터미널에서 입력 (매번)
- ❌ API 키를 누구와도 공유 금지
- ✅ IP 제한 설정 필수

### 5.2 거래
- 초기 베타는 **50만원 이하** 권장
- 1주 테스트 후 확대 고려
- **약세장은 거래 중단** (손실 회피)
- 손절 기준 -5% **절대 완화 금지**

### 5.3 모니터링
- 최소 1일 1회 상태 확인
- 극단 악재 발생 시 수동 개입
- 규제 발표 시 즉시 손절

### 5.4 시스템
- Mac 24시간 켜둬야 함 (슬립모드 OFF)
- 네트워크 안정 필수
- 메모리 부족 시 프로그램 종료 후 재시작

## 📚 6. FAQ

### Q1: 데모 모드는 어떻게 작동하나?
**A:** 실제 API를 호출하되, 자금은 이동하지 않습니다.
- 가상 500만원으로 시뮬레이션
- JSON 로그에 거래 기록됨
- 실제 가격으로 수익률 계산

### Q2: 실제 모드로 바꾸려면?
**A:** 프로그램 시작 시 `mode: real` 선택
```bash
모드 선택 (demo/real): real
초기 자금 (원): 500000
Access Key: your_access_key
Secret Key: your_secret_key
```

### Q3: 손실이 계속 나면?
**A:** 
1. 약세장인지 확인 (시장 상황)
2. 초기 설정 검토
3. 손절 기준 확인 (-5%)
4. 1주 더 테스트 후 판단

### Q4: 로그는 어디에 저장되나?
**A:** 실행 디렉토리에 저장됨
- `trading_log_YYYYMMDD_HHMMSS.json` 형식
- 거래 기록, 수익률, 포지션 정보 포함

### Q5: 포지션을 강제로 매도하고 싶은데?
**A:** 현재 코드에서 미지원. 수동으로:
```bash
# 업비트 앱에서 직접 매도
# 또는 추후 'sell BTC' 명령어 추가
```

### Q6: 여러 코인을 동시에 거래하고 싶은데?
**A:** 현재는 최대 2개 코인만 (설정에 따라)
- 더 많은 코인 추가는 포트폴리오 분산화 필요

### Q7: 일론머스크 트윗이 감시되나?
**A:** 현재 기본 구현은 뉴스 API만.
- SNS 실시간 감시는 추후 업그레이드 예정

### Q8: 약세장이란 뭔가?
**A:** 전체 시장이 하락하는 시기
- 모든 코인이 함께 내려가는 상황
- 이때는 최대한 거래를 피함
- "덜 떨어지는 것"만 캐치
