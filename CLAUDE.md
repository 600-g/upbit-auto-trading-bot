## 소통 규칙

- 회사 동료에게 메신저로 말하듯 자연스럽고 친근하게 대화해.
- 코드 블록은 꼭 필요할 때만 짧게 보여주고, 나머지는 자연어로 설명해.
- 두근이 두루뭉술하게 말해도 의도를 파악해서 실행해.
- 애매한 요청은 "이렇게 이해했는데 맞나요?" 한 줄 확인 후 바로 실행해.
- 80% 이상 확신이면 실행 후 보고하고, 지나치게 되묻지 마.
- "안돼요" 대신 "이건 안 되는데, 이렇게 하면 돼요" 식으로 대안을 줘.
- 전문 용어(매수/매도/RSI/볼린저밴드 등)는 항상 쉽게 풀어서 설명해.
- 작업 완료 시 "✅ 완료했습니다. (결과 요약)" 한 줄로 알려줘.
- 진행 중에는 뭘 하고 있는지 간단히 알려줘.

## 핵심 설정 위치

| 설정 | 파일/위치 | 설명 |
|------|----------|------|
| 손절라인 | `monitor_positions()` → `stop_loss` | -5% 절대 완화 금지 |
| 익절라인 | `monitor_positions()` → `target` | 수익 목표 |

| 최대 보유 | `MAX_POSITIONS` | 동시 보유 종목 수 |
| 루프 주기 | 메인 루프 `sleep` | 체크 간격 |

## 봇 제어 명령

```bash
# 봇 상태 확인
ps aux | grep upbit_bot

# 봇 중지
kill $(cat bot.pid) 2>/dev/null

# 봇 시작
nohup ./venv/bin/python -u upbit_bot_v3_0_complete.py demo --auto >> bot_output.log 2>&1 &
echo $! > bot.pid

# 최근 로그 확인
tail -50 bot_output.log
```

## 매매 판단 프로세스 (멀티 페르소나 + 토론 + 기억)

모든 매매 판단(매수/매도/종목탐색)에 아래 시스템을 적용해.
**자동매매 실행 전**에도 반드시 이 프로세스를 거쳐야 해.

---

### Phase 1: 정보 수집 (4관점 동시, 하나의 분석)

아래 4가지 관점을 **한번에** 분석해 (순차 X, 동시 사고):

| 관점 | 체크 항목 | 결론 형식 |
|------|----------|----------|
| 🔍 기술 | RSI, MACD, 볼린저, 이평선, 패턴, 거래량 | 신호 [매수/매도/관망] 강도 1~10 |
| 📰 매크로 | 호재/악재, 글로벌시장, 규제, ETF | 환경 [우호/비우호/중립] 강도 1~10 |
| 💬 감성 | 업비트 거래량, 김프, 공포탐욕 | 심리 [탐욕/공포/중립] 강도 1~10 |
| 📊 모멘텀 | 수익률 상위, 거래량 급증, 이슈 코인 | 주목 종목 [리스트] |

---

### Phase 2: 토론 (1라운드, 핵심만)

하나의 흐름으로 강세/약세를 **대비**:

```
🐂 강세: "(Phase 1 근거) → 매수해야 하는 이유, 목표가 X원"
🐻 약세: "(강세 반박) → 안 되는 이유, 최악 시 Y원 손실"
🐂 재반론: "(약세 약점) → 그래도 사야 하는 핵심 1가지"
```

> 규칙: 상대 주장 인용+반박 필수. 애매한 동의 금지. 수치로 근거.

---

### Phase 3: 리스크 + 기억 + 최종 판단 (통합)

**리스크 심의:**
- 🔥 공격파 vs 🛡 보수파 → 비율 결정
- 현재 보유 수, 잔고, 연속 손실 고려

**과거 기억** (`trade_memory.json`):
- 유사 상황 검색 → 그때 판단 + 결과 참고
- 같은 실수 반복 방지
- 없으면 "유사 사례 없음"

**최종 판단:**

```
═══════════════════════════════════
⚖️ 최종 판단
───────────────────────────────────
판정: BUY / SELL / HOLD
종목: (코인명)
비율: 투자금의 X%
진입가: 현재가 기준
손절: -X% (절대 완화 금지)
익절: +Y% (1차), +Z% (2차)
───────────────────────────────────
근거: (한 줄)
강세 점수: X/10 | 약세 점수: Y/10
리스크 등급: 낮음/중간/높음
과거 유사 사례: (있으면 결과 포함)
═══════════════════════════════════
```

---

### Phase 4: 매매 후 기억 저장

매수 또는 매도 **실행 후** 아래를 `trade_memory.json`에 추가:

```json
{
  "date": "2026-03-21",
  "coin": "BTC",
  "action": "BUY",
  "price": 85000000,
  "reason": "RSI 과매도 + ETF 호재",
  "bull_score": 7,
  "bear_score": 4,
  "risk_level": "중간",
  "result": null
}
```

→ 이후 해당 포지션 정리 시 `result`에 수익률 기록
→ 다음 판단 시 Phase 4에서 참조

## Git 규칙 (필수 — 자동 커밋)

**코드 수정 후 반드시 커밋+푸시해야 한다. 예외 없음.**

작업 완료 시 아래를 자동 실행:
```bash
git add .
git commit -m "feat/fix/refactor: 한글 작업 내용 요약"
git push
```

커밋 타입: feat, fix, refactor, docs, config, chore

⚠️ 커밋 안 하면 다른 에이전트/세션에서 작업이 유실됨
⚠️ 큰 작업은 중간중간 커밋 (한번에 몰아서 X)

## 자가 발전 루프 (lessons.md)

실수나 수정이 발생하면 패턴을 기록하여 같은 실수를 반복하지 않는다.

- 프로젝트 루트에 `lessons.md` 파일 유지
- 형식: `[날짜] 문제 → 원인 → 재발 방지 규칙`
- 새 대화 시작 시 lessons.md를 읽고 과거 실수를 인지한다
- 같은 실수 2회 반복 시 CLAUDE.md 본문에 규칙으로 승격

## 완료 전 필수 검증

코드 수정 후 "작동한다"는 증거 없이 완료 보고하지 않는다.

- 코드 수정 → 빌드/테스트 성공 확인 필수
- 빌드 실패 시 완료 보고 금지, 즉시 수정 루프 진입
- "이렇게 확인해봐" 가이드 필수 제공
- 검증 통과 후에만 ✅ 완료 보고

---

## 계획 우선 원칙

3단계 이상의 복잡한 작업은 계획 → 검증 → 실행 순서를 지킨다.

- 단순 작업(1~2단계)은 바로 실행 OK
- 복잡한 작업은 먼저 "이렇게 할게" 목록을 보여주고 진행
- 계획 변경 시 변경 사항을 먼저 공유
- 완료 후 전체 결과 요약 보고

## 공통 규칙 (company-hq v2.0)

### 디스패치 협업
- CPO 또는 다른 팀에서 디스패치 작업이 올 수 있다
- 디스패치 작업을 받으면 본인 역할 범위 내에서 수행하고 결과를 텍스트로 반환
- 본인 담당이 아닌 작업이면 '⏭ 해당없음' 한 줄만 답변

### 보안 규칙
- API Key, 토큰, 비밀번호는 채팅에 절대 노출 금지
- .env 파일 내용은 로그/채팅/커밋에 포함 금지
- 파일 삭제, 덮어쓰기 전 반드시 확인

### 에러 대응
- 확실하지 않으면 "확실하지 않다"고 솔직히 말한다
- 모르면 "모르겠다"고 말한다 (거짓 답변 금지)
- 해결 후 "이게 보이면 성공" 확인 방법 제시

### 응답 규칙
- CLAUDE.md 최우선. 무응답 금지
- 완료 → ✅ 한 줄 요약
- 에러 → ❌ 에러 내용
- 한국어로 자연스럽게 대화
