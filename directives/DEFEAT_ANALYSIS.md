# 패배 분류 & 자가개선 기준서
> 위치: ~/Desktop/업비트자동/directives/DEFEAT_ANALYSIS.md
> 목적: 고칠 수 있는 패배만 골라서 줄이기 → 승률 자연 상승 → 월 10~20%
> 페르소나: 5인 합의체 (김트레이드 팀장)

---

## 🎯 핵심 철학

```
패배는 두 종류다.

Type A. 어쩔 수 없는 패배  → 기록만, 개선 X
Type B. 고칠 수 있는 패배  → 반드시 자가개선

Type B만 줄이면 승률은 자연스럽게 오른다.
Type A까지 막으려다가 진입 자체를 못 하게 된다.
```

---

## 📊 패배 분류표

### Type A: 어쩔 수 없는 패배 (개선 대상 아님)

| 패턴 | 설명 | 판단 기준 |
|------|------|-----------|
| 시장 급변 | BTC 갑작스러운 -3% 이상 | 진입 당시 신호는 정상이었음 |
| 뉴스/이벤트 | 예측 불가 외부 충격 | 기술적 분석으로 알 수 없었음 |
| 유동성 부족 | 슬리피지로 손절 미체결 | 코인 자체 문제 |
| 정상 손절 | 손절 라인 건드리고 반등 | 규칙대로 실행됨, 결과만 나쁜 것 |

> ✅ Type A 발생 시: 로그 기록 후 넘어감. 패치 금지.

---

### Type B: 고칠 수 있는 패배 (반드시 자가개선)

| 패턴 | 설명 | 심각도 |
|------|------|--------|
| **분석 실패** | 진입 조건 충족했지만 근거가 약했음 | 🔴 높음 |
| **매도 타이밍 오류** | 수익 구간에서 너무 일찍/늦게 매도 | 🔴 높음 |
| **과진입** | 모멘텀 약한데 진입 기준 낮춰서 들어감 | 🟡 중간 |
| **반복 실수** | 같은 패턴 실수 2회 이상 | 🔴 높음 |
| **물타기 오판** | 반등 신호 없이 물타기 실행 | 🔴 높음 |
| **쿨다운 무시** | 연속 손실 후 바로 재진입 | 🟡 중간 |

> ❌ Type B 발생 시: 즉시 소크라테스 복기 → 개선안 도출 → 백테스트 → 적용

---

## 🔄 자가개선 실행 기준

### 개선 트리거 (Type B 확인 시)

```
즉시 개선:
- 같은 Type B 패턴 하루 3회 이상 반복
- 매도 타이밍 오류로 수익 → 손실 전환
- 분석 실패로 -2% 이상 단일 손실

다음 사이클 개선:
- Type B 1~2회 산발적 발생
- 경미한 과진입

개선 보류 (데이터 부족):
- 30회 미만 샘플
- 단 1회 발생, 재현 여부 불명확
```

---

### 개선 프로세스 (5인 합의체 역할)

```
Step 1. 김트레이드 (팀장)
  → "이게 Type A야 Type B야?" 1차 분류

Step 2. 이퀀트 (알고)
  → 같은 패턴 과거 발생 횟수 확인
  → "재현율 몇 %?" 숫자로만 판단

Step 3. 해당 전문가 분석
  → 매도 오류 → 최스캘퍼 주도
  → 분석 실패 → 박터틀 주도
  → 리스크 오판 → 정디펜더 주도

Step 4. 개선안 도출
  → 변수 1개만 변경 (동시에 여러 개 금지)
  → 백테스트 30회 이상 검증

Step 5. 적용 & 모니터링
  → 48시간 후 재평가
  → 개선 미확인 시 롤백
```

## 📉 매도 타이밍 오류 판단 기준

가장 빈번한 Type B. 구체적 기준 필요.

```
조기 익절 (early_exit) 판단:
- 매도 후 10분 내 +1% 이상 추가 상승 → 조기 익절로 기록
- 원인: 익절 타겟이 너무 낮거나 트레일링 미작동

늦은 손절 판단:
- 손절 라인 돌파 후 체결까지 0.3% 이상 추가 하락 → 지연 손절로 기록
- 원인: 슬리피지 or 손절 로직 오류

수익 반납 판단:
- +2% 이상 찍고 손절로 마감 → 수익 반납으로 기록
- 원인: 트레일링 설정 문제
```

## 🧠 페르소나별 패배 분석 관점

### 🎯 김트레이드 (팀장, 모멘텀)
```
체크: 진입 당시 거래량+RSI가 기준 충족했는가?
충족했으면 → Type A 가능성
미충족했으면 → Type B (과진입)
```

### 🧮 이퀀트 (알고)
```
체크: 같은 조건 과거 승률은?
55% 이상이면 → 이번만 운 나쁜 것 (Type A)
55% 미만이면 → 전략 자체 문제 (Type B)
```

### 🐢 박터틀 (보수)
```
체크: 큰 추세에 역행한 진입이었는가?
BTC 하락 중 알트 롱 → Type B (시장 읽기 실패)
추세 맞았는데 손실 → Type A
```

### ⚡ 최스캘퍼 (단타)
```
체크: 매도 타이밍이 문제인가?
고점 대비 30% 이상 내려서 팔았으면 → Type B
트레일링 정상 작동했으면 → Type A
```

### 🛡 정디펜더 (리스크)
```
체크: 이 손실이 예상 범위 내인가?
단일 손실 -2% 이내 → 정상 범위 (Type A)
-2% 초과 → 리스크 관리 실패 (Type B)
```

## 📈 승률 자연 상승 구조

```
Type B 패배 발생
    ↓
5인 분류 → 개선안 도출
    ↓
변수 1개 수정 → 백테스트
    ↓
Type B 해당 패턴 감소
    ↓
전체 승률 상승
    ↓
월 수익률 10~20% 달성
```

Type A는 줄이려 하지 않는다.
Type A를 막으려다가 진입을 못 하게 되는 게 더 큰 손실이다.

## 🚫 절대 하지 말 것

```
1. Type A 발생했다고 파라미터 바꾸지 않는다
2. 30회 미만 샘플로 전략 삭제하지 않는다
3. 하루에 2개 이상 변수 동시에 바꾸지 않는다
4. 데이터 없이 "느낌상 이게 문제인 것 같다"로 패치하지 않는다
5. 패치 후 48시간 안에 또 패치하지 않는다
```

## 🎯 월 10~20% 달성 체크포인트

```
1주차: Type B 분류 정확도 확인 (제대로 잡고 있는가)
2주차: Type B 패턴 감소 확인 (개선이 효과 있는가)
3주차: 승률 50% → 55% 도달 확인
4주차: R:R 1.5 이상 + 승률 55% = 월 10~20% 달성 구조
```
